Proposta

Para este desafio, vamos nos aprofundar nos conhecimentos adquiridos em aula para construir uma base de dados e uma API responsáveis pelo cadastro e leitura de funcionários(as) de uma empresa. 
Então, considere o cenário a seguir. 

Somos uma empresa especializada em recrutamento e seleção e temos um sistema que lista todas as pessoas que contratamos e conta com a possibilidade de inserir novos registros. 
Além disso, nosso sistema abrange os dados pessoais de cada um desses indivíduos, assim como os salários oferecidos.

Dado o cenário descrito, devemos garantir que nossa API cadastre e retorne os dados do nosso BD, que deverá possuir a seguinte estrutura:

1. Uma tabela com os dados básicos dos(as) funcionários(as), como: nome, idade, nome dos pais.
2. Uma tabela com seus endereços: id do(a) funcionário(a), rua, número, cep, cidade e estado.
3. Uma tabela com as informações de entrevistas: empresa, data, salário, responsável pela vaga.
   
Orientações:

1. Crie um projeto no template padrão de APIs e utilize o Dapper como ORM responsável para extração e inserção das informações.
2. Crie as pastas necessárias para organizar seu projeto.
3. Salve todas as queries criadas para a entrega do trabalho.
4. Foque em resolver o problema proposto e depois pense em maneiras de evoluir sua aplicação.
